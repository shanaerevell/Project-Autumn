HiddenLeaf Studios Organization Structure

Purpose:

This document defines the organizational structure, departments, user roles, and security groups used within Project Autumn.

The goal is to simulate a realistic enterprise environment where access is granted based on job responsibilities and the principle of least privilege.



Departments:

Executive Leadership

Responsible for organizational strategy and oversight.

Roles:

* Chief Executive Officer (CEO)
* Chief Technology Officer (CTO)



Information Technology

Responsible for infrastructure, systems administration, networking, security, and user support.

Roles:

* Systems Administrator
* Cloud Administrator
* Help Desk Technician
* Network Administrator



Development

Responsible for application development and maintenance.

Roles:

* Software Developer
* Web Developer
* QA Tester



Operations

Responsible for daily business operations and coordination.

Roles:

* Operations Manager
* Operations Coordinator



Active Directory Security Groups:

* IT Groups
* HLS-IT-Admins

Purpose:
Administrative access to servers and infrastructure systems.

Members:

* Systems Administrator
* Cloud Administrator



HLS-HelpDesk

Purpose:
User support and workstation administration.

Members:

* Help Desk Technician



Development Groups

HLS-Developers

Purpose:
Access to development resources and repositories.

Members:

* Software Developers
* Web Developers



Operations Groups

HLS-Operations

Purpose:
Access to operational resources and shared documentation.

Members:

* Operations staff



Administrative Accounts:

Administrative privileges should be separated from standard user accounts.

Example:

Standard Account:

* shanae.revell

Administrative Account:

* shanae.admin

This follows enterprise security best practices.
